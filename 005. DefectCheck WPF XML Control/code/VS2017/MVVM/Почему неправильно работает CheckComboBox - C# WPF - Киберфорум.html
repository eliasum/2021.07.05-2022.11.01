     Комментарий модератора 
    При обращении к другому пользователю указывайте его ник в тегах [NICK][/NICK] или цитируйте часть сообщения на которое отвечаете.
    В противном случае ему не придёт уведомление о вашем обращении и вы можете не дождаться ответа на своё сообщение.

    Для вставки ника: введите ник, выделите его и нажмите кнопку "Динамик" на панели редактора сообщений.
    Или кликните по нику автора сообщения в панели слева от текста его сообщения.

    Для вставки цитаты: выделите нужную цитату, должна появиться всплывающая кнопка "Цитировать", нажмите её.

    eliasum, не приведя в порядок структуру данных делать для них представление бессмысленно - всё равно приложение будет работать криво.

    Поэтому сначала давайте займёмся сами данными.

    1) <Product title="Проверяемое изделие"> - это root элемент?
    То есть такое изделие одно, не предусматривается работа с несколькими в одном XML файле?
    Или есть более высокий уровень иерархии в котором находятся несколько <Product>?

    2) <Settings title="Изделие"> зачем этот ЕДИНСТВЕННЫЙ тег внутри <Product>?
    Какова его информативность?
    Или в <Product> могут быть ещё иные теги?

    3) <Defect> и <Type> - такой же вопрос: зачем в <Defect> единственный тег <Type>?

    4) Список <Item> в <Defect><Type> - насколько они уникальны?
    В такой как у вас структуре подразумевается, что они уникальны для каждого <Product>.

    5) Это не вопрос а рекомендации:
    - <Item key="01" такие уникальные ключи принято называть "id".
    Это упрощает понимание кода для других.

    - Само название Item тоже очень малоинформативно.
    Нужно название отражающее суть, назначение хранимой информации.
    Если это настройка, то Setting. Если дефект, то Defect.
