Первое задание: создать XSLT правило, которое «сливает» оба файла configuration.xml и profile.xml
в один файл configuration+profile.xml таким образом, чтобы в дереве configuration.xml в каждом теге 
<Chart> появился соответствующий дочерний тег <Widget> из файла profile.xml. 

Примечание:
Не должно быть привязки к тегу <Chart>, т.е. в выходной файл сливается всё содержимое файла 
configuration.xml, производится сравнение с файлом profile.xml (т.к. они идентичны по узлам, 
являющимися предками для Widget) и, когда доходим до тега <Widget>, добавляем его в выходной файл).
